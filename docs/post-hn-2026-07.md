**First comment** (post it immediately, from the same account):

I built this because the live picture was never what I was missing. The past
was. Flightradar24's free tier gives you 7 days of history, MarineTraffic cut
its free window from 72 to 24 hours, ADS-B Exchange discontinued its free API
tier. Every time I wanted "where was this vessel last Tuesday" the answer was a
subscription. A box in my house doesn't have that problem, so it records the
picture and keeps it until my disk fills up.

What it is: a FastAPI backend and a Cesium globe that fuse aircraft (OpenSky
plus the airplanes.live community grid), vessels (keyless AIS, MMSI-deduped
across ShipXplorer and the Norway/Baltic regional feeds), satellites (CelesTrak
TLEs, client-side SGP4), earthquakes, wildfires, GPS jamming inferred from
ADS-B NACp/NIC degradation, and conflict events. One `docker compose up`, no
API key for any core feed.

Right now on the box I'm typing this on: 21,186 aircraft and 55,086 vessels in
the live snapshot, refreshed 0.2 s ago; 8.7M archived positions in a 1.8 GB
SQLite file you can open with the sqlite3 CLI. Retention is a size cap you set
(this box's is 2 GB; it holds the oldest slice under that and keeps the rest).

The engineering that actually took the time:

- Feed politeness. adsb.lol 451s non-browser user agents. airplanes.live
  throttles by returning HTTP 200 with a text/plain body, so a naive JSON parse
  swallows it as an empty world. CelesTrak rate-limits bursts with 403. Most of
  the backend's scar tissue is upstreams failing in ways that look like success.
- Keeping ~15k aircraft at 60 fps. Cesium entities died around 3k, so
  aircraft/vessels/satellites render as batched primitives with
  `requestRenderMode` on. The globe redraws on data, not on a timer.
- Not lying about motion. Aircraft teleport to real fixes by default. Predicted
  motion is opt-in and analytic: `pos(t) = advance(anchor, track_deg,
  velocity_ms * dt)` from the reported speed. I tried fitting velocity from
  consecutive fixes first; only 13.5% of fits landed within ±10% of the
  reported value, and interpolating toward the next fix makes aircraft glide
  backwards when a stale fix arrives. A tracker that invents positions is worse
  than one that jumps.
- Freshest-observation-wins across tiers. `seen_pos_s` is the age at upstream
  serve time, so a cached tier reports its data as fresh forever. Comparing it
  across tiers lets a cache clobber a live fix.

Honest caveats: coverage is community feeders, so it's dense over Europe and
the US and thin over open ocean and conflict zones, which is exactly where
you'd want it. AIS is strongest in Northern Europe. It's a single-analyst tool,
not a team server. The 3D globe wants a real GPU. There are AI summarization
features; they're optional, labeled as automated output, and run against local
inference if you point it at a local model. No public demo instance for the
feeds, because redistribution isn't what the community feeders signed up for.

Feedback I'd genuinely use: which keyless sources are missing that you'd
actually run, and whether the evidence export (SHA-256 custody log, cases out
as self-contained HTML/PPTX) holds up against how you'd document something.
